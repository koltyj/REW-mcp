Create `src/interpretation/glm-comparison.ts` implementing GLM comparison logic:

**1. Types and interfaces:**
```typescript
export interface GLMComparisonResult {
  glm_successes: GLMCorrection[];
  glm_persistent: GLMBeyondScope[];
  glm_observations: Array<{
    observation: string;
    explanation: string;
    is_expected: boolean;
  }>;
  overcorrection_indicators: {
    bass_flatness: { detected: boolean; variance_db: number; threshold_db: number };
    null_revelation: { detected: boolean; contrast_increase_db: number };
  };
  mode: 'full_comparison' | 'post_only_heuristic';
  confidence: ConfidenceLevel;
}

interface IssueComparison {
  frequency_hz: number;
  pre_deviation_db: number;
  post_deviation_db: number;
  improvement_db: number;
  improvement_percent: number;
}
```

**2. Core comparison function:**
```typescript
export function compareGLMCalibration(
  preMeasurement: StoredMeasurement,
  postMeasurement: StoredMeasurement
): GLMComparisonResult
```

Implementation:
- Call detectPeaks/detectNulls on both measurements
- Match peaks/nulls by frequency (within 5 Hz tolerance)
- For each matched issue, calculate IssueComparison
- Classify using proportional threshold (50%+ reduction = success)
- Use context-dependent unchanged thresholds:
  - Small issues (<6 dB): <1 dB change = unchanged
  - Medium issues (6-10 dB): <2 dB change = unchanged
  - Large issues (>10 dB): <3 dB change = unchanged

**3. Classification function:**
```typescript
function classifyCorrection(comparison: IssueComparison): 'success' | 'partial' | 'unchanged'
```

**4. GLM success mapping:**
For issues classified as 'success':
- Create GLMCorrection object with effectiveness based on improvement_percent:
  - 75%+ = 'highly_effective'
  - 50-75% = 'effective'
  - 25-50% = 'partially_effective'
  - <25% = 'minimal_effect'
- Include explanation like "Peak at {freq} Hz reduced from {pre} dB to {post} dB ({percent}% improvement)"

**5. GLM persistent mapping:**
For deep nulls (>10 dB depth) in post measurement:
- Check if null existed pre-GLM at similar depth (use classifySBIR for SBIR classification)
- Create GLMBeyondScope object with:
  - why_glm_cannot_fix.reason = 'cut_only_correction' for nulls
  - why_glm_cannot_fix.explanation = "GLM applies cuts only, never boosts. Deep nulls cannot be filled."
  - Include recommended_solutions with placement suggestions

**6. GLM observations:**
- Note when peaks reduced significantly (GLM cut filters working)
- Note when bass variance improved (smoother response)
- Note expected behaviors (nulls unchanged - as designed)

**7. Overcorrection detection with comparison:**
```typescript
function detectOvercorrectionWithComparison(
  preMeasurement: StoredMeasurement,
  postMeasurement: StoredMeasurement
): { bass_flatness: {...}; null_revelation: {...} }
```

- Bass flatness: Check variance below 40 Hz in post measurement (<2 dB = unnaturally flat)
- Null revelation: Compare null depths pre vs post - if nulls appear deeper post-GLM by >3 dB (because surrounding peaks reduced), flag as contrast increase

- confidence: 'medium' (heuristics, not definitive)

**2. Overcorrection detection (post-only):**
```typescript
export function detectOvercorrection(postMeasurement: StoredMeasurement): {

  null_revelation: { detected: boolean; contrast_increase_db: number };
}
```

- Bass flatness: Extract SPL values for 20-40 Hz range, calculate variance
  - detected = variance < 2 dB
- Null revelation: Cannot detect in post-only mode (requires comparison)

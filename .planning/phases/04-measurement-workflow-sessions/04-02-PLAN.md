<task type="auto">
  <name>Task 1: Create sequence state machine module</name>
  <files>src/session/sequence-state-machine.ts</files>
  <action>
Create state machine module with:

1. SequenceStep type:
   - 'idle' (session created, no measurements)
   - 'measuring_left' (left measured)
   - 'measuring_right' (right measured)
   - 'measuring_sub' (sub measured)
   - 'complete' (all done)

2. validTransitions constant (Record<SequenceStep, SequenceStep[]>):
   - idle -> ['measuring_left']
   - measuring_left -> ['measuring_right']
   - measuring_right -> ['measuring_sub']
   - measuring_sub -> ['complete']
   - complete -> [] (no further transitions)

3. validateTransition(from: SequenceStep, to: SequenceStep): void
   - Check if transition is in validTransitions[from]
   - If not, throw Error with message:
     "Invalid measurement sequence: Cannot go from '{from}' to '{to}'. Expected next step: {validTransitions[from].join(' or ') || 'none (sequence complete)'}"

4. getNextStep(current: SequenceStep): SequenceStep | null
   - Return first valid transition or null if complete

5. getStepGuidance(step: SequenceStep): string
   - idle: "Ready to start. Measure Left speaker first."
   - measuring_left: "Left speaker measured. Measure Right speaker next."
   - measuring_right: "Left and Right measured. Measure Subwoofer next."
   - measuring_sub: "All speakers measured. Sequence complete."
   - complete: "Measurement sequence complete. Use stop_session to end."

6. channelToStep(channel: 'left' | 'right' | 'sub'): SequenceStep
   - Helper to convert channel to step (for transition validation)
   - left -> 'measuring_left'
   - right -> 'measuring_right'
   - sub -> 'measuring_sub'

Use Record type for validTransitions (type-safe, exhaustive).

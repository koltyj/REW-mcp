  <name>Task 1: Create measurement session tool</name>
  <files>src/tools/api-measurement-session.ts</files>
  <action>
Create MCP tool implementing the measurement session workflow:

1. Input schema (ApiMeasurementSessionInputSchema):
   - action: z.enum(['start_session', 'measure', 'get_status', 'stop_session'])
   - session_id: z.string().uuid().optional() - required for measure/get_status/stop_session
   - channel: z.enum(['left', 'right', 'sub']).optional() - required for measure action
   - notes: z.string().optional() - for start_session

2. Result interface (ApiMeasurementSessionResult):
   - action: string
   - session_id?: string
   - sequence_step?: SequenceStep
   - next_step?: string | null
   - message: string
   - guidance?: string
   - measurements?: SessionMeasurement[]
   - session?: SessionState (for get_status)
   - active_sessions?: SessionState[] (for get_status without session_id)

3. executeApiMeasurementSession function:

   start_session action:
   - Reject if session_id provided (must omit for new session)
   - Call createSession(notes)
   - Return session_id, sequence_step: 'idle', guidance

   measure action:
   - Require session_id and channel
   - getSession (throws if not found - handle with error response)
   - channelToStep to get target step
   - validateTransition (throws if invalid - handle with error response)
   - Generate measurement name: `${session.session_id.slice(0, 8)}_${channel}`
   - setMeasureNotes(name)
   - setBlockingMode(true)
   - Get beforeCount via listMeasurements().length
   - executeMeasureCommand('Measure')
   - Handle 403 status: "REW Pro license required for automated measurements"
   - Handle other errors
   - Get afterMeasurements via listMeasurements()
   - Find new measurement (last in list if count increased)
   - Add measurement to session.measurements
   - Update session.sequence_step
   - If channel === 'sub', set sequence_step to 'complete'
   - updateSession
   - Return measurement details, next_step, guidance

   get_status action:
   - If session_id provided: getSession and return full session + guidance
   - If no session_id: listActiveSessions and return count + list

   stop_session action:
   - Require session_id
   - getSession (handle not found)
   - endSession
   - Return measurements_taken count, sequence_complete flag

4. Error handling:
   - Zod validation errors -> validation_error
   - Session not found -> session_error with suggestion
   - REW not connected -> connection_error
   - 403 from measure -> license_error with upgrade URL
   - REWApiError -> map to appropriate error type

Follow api-calibrate-spl.ts structure and error handling patterns exactly.

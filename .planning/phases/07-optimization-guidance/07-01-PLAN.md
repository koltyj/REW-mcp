Create `src/optimization/recommendations.ts` with:

1. **generatePlacementRecommendation(issue: PrioritizedIssue): PlacementRecommendation**
   - For SBIR issues: Use quarter-wavelength formula from classifySBIR
   - For room modes: Suggest position adjustment to avoid mode nulls
   - For peaks: Suggest boundary distance changes
   - For nulls: Suggest repositioning (not EQ - can't boost)
   - Include brief physics context in reason field
   - Use confidence based on issue category:
     - SBIR with Q>10: 'high'
     - SBIR with Q>5: 'medium'
     - Room modes with dimensions: 'medium'
     - Room modes without dimensions: 'low'
     - Other: 'medium'

2. **generateSubRecommendation(issue: PrioritizedIssue, phaseInfo?: PhaseInfo): SubRecommendationDetail**
   - More detailed than monitors per CONTEXT.md
   - Include phase_suggestion if phase_inversion detected
   - Include boundary_loading context ("Corner placement adds +3dB per boundary")
   - Include crossover_context if issue is near typical crossover (60-100 Hz)
   - Reference corner loading physics (+3 dB per boundary)

3. **generateListeningPositionRecommendation(issue: PrioritizedIssue, roomDimensions?: RoomDimensions): PlacementRecommendation**
   - Use 38% rule when dimensions available ("Move to ~38% of room length")
   - For mode nulls at L/4, L/2, 3L/4: Suggest avoiding null zones
   - Without dimensions: Generic "try moving forward/backward" guidance
   - Confidence based on dimensions availability

4. **Helper: determineElement(issue: PrioritizedIssue): OptimizationElement**
   - SBIR: 'monitors'
   - sub_integration: 'subwoofer'
   - lr_symmetry: 'monitors'
   - room_modes below 80Hz: 'subwoofer' or 'listening_position'
   - room_modes 80-200Hz: 'monitors' or 'listening_position'

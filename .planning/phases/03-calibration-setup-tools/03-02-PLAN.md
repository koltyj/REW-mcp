Level zone determination function:
```typescript
type LevelZone = 'CLIPPING' | 'HOT' | 'OPTIMAL' | 'LOW' | 'VERY_LOW';

function determineLevelZone(rms: number, peak: number): LevelZone {
  // Priority: Clipping first (SETV-02)
  if (peak > -3) return 'CLIPPING';

  // RMS zones (SETV-01, SETV-03)
  if (rms > -10) return 'HOT';
  if (rms >= -20 && rms <= -10) return 'OPTIMAL';  // Target: -12 dBFS
  if (rms >= -40 && rms < -20) return 'LOW';

  return 'VERY_LOW';  // < -40 dBFS (SETV-03)
}
```

Feedback formatting function:
```typescript
function formatLevelFeedback(zone: LevelZone, rms: number, peak: number, targetRMS: number): string {
  // Zone display with emoji indicators
  const zoneDisplay: Record<LevelZone, string> = {
    CLIPPING: `Warning: CLIPPING (${peak.toFixed(1)} dBFS peak)`,
    HOT: `Hot signal (${rms.toFixed(1)} dBFS RMS)`,
    OPTIMAL: `Good range (${rms.toFixed(1)} dBFS RMS)`,
    LOW: `Low signal (${rms.toFixed(1)} dBFS RMS)`,
    VERY_LOW: `Warning: VERY LOW (${rms.toFixed(1)} dBFS RMS)`
  };

  // Guidance per zone (SETV-04)
  const guidance: Record<LevelZone, string> = {
    CLIPPING: 'Turn down mic preamp gain, then check again',
    HOT: 'Turn down mic preamp gain slightly, then check again',
    OPTIMAL: `Ready for measurement (target: ${targetRMS.toFixed(0)} dBFS RMS)`,
    LOW: 'Increase mic preamp gain, then check again',
    VERY_LOW: 'Increase mic preamp gain significantly, then check again. Check mic is connected and phantom power is enabled if needed.'
  };

  return `${zoneDisplay[zone]}\n${guidance[zone]}`;
}
```

Result interface:
```typescript
export interface ApiCheckLevelsResult {
  action: string;
  zone: LevelZone;
  rms_db: number;
  peak_db: number;
  should_block_measurement: boolean;  // true if CLIPPING or VERY_LOW
  feedback: string;
  channels: Array<{
    channel: number;
    rms_db: number;
    peak_db: number;
  }>;
}
```

Main execute function:
1. Validate input with Zod
2. Get active API client (return connection error if null)
3. Start input level monitoring
4. Get input levels (return error if null - monitoring not started or audio device issue)
5. Calculate avgRMS (average across channels) and maxPeak (max across channels)
6. Determine zone
7. Check for L/R mismatch: if 2 channels and diff > 3 dB, add warning to feedback
8. Stop input level monitoring in finally block
9. Return result with zone, levels, should_block_measurement, and feedback

Error handling: Follow same pattern as api-generator.ts (ZodError, REWApiError, generic Error).
